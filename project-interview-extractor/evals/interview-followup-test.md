# Eval：模拟面试连续追问

## 初始输入

```text
进入模拟面试模式。我负责 React 管理后台的订单列表，包括筛选、分页和导出。
```

## 用户第一轮回答样例

```text
页面加载时请求订单接口，筛选条件变更后重新请求。为了避免慢，我用了 useMemo。
```

## 预期行为

- 初始回复只问一个最贴近职责的问题。
- 用户回答后指出 `useMemo` 与接口请求/网络性能之间的关系尚未讲清。
- 下一问应追踪实际瓶颈、测量方法、请求竞态或筛选状态，不突然切到无关的微服务题。
- 反馈简短，给出评分依据，但不在同一轮抛出多个问题。
- 不把使用 `useMemo` 自动表述为“显著优化页面性能”。

## 完成条件

覆盖事实核验、原理深挖、约束变化三层后，再切换到导出、错误状态、可访问性或测试等相邻主题；用户结束时给综合评价和保守版改写。
