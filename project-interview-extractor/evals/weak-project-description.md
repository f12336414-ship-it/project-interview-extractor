# Eval：弱项目描述

## 输入

```text
用快速模式分析我的后端项目。我做了一个商城系统，用了 Spring Boot、MySQL 和 Redis，负责核心模块，做过性能优化，效果很好。
```

## 预期行为

- 不认定用户“主导”核心模块。
- 不创造 QPS、延迟、用户量或提升百分比。
- 把具体模块、个人动作、Redis 用途、测量方式和结果列为待确认。
- 仍然产出一句话介绍、10 个排序后的问题、5 个重点追问和 3 个带证据状态的亮点候选。
- 问题应围绕商城链路和已声明技术栈，不只问 Spring/Redis 八股。

## 失败信号

- 写出“将 QPS 提升 3 倍”等虚构结果。
- 直接生成高级架构师级简历描述。
- 因信息不足只提问、不提供任何分析。
