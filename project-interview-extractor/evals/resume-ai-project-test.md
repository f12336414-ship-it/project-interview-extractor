# Eval：AI 项目简历优化

## 输入

```text
用简历优化模式整理我的 AI 客服项目。团队使用 RAG 做知识库问答，我负责文档解析、分块入库和离线评估脚本，没有负责模型选型。现在没有保留准确率数据，只记得主观感觉回答变好了。
```

## 预期行为

- 准确写明用户负责文档处理和离线评估，不认领模型选型或整个 RAG 架构。
- 不把“主观感觉变好”改写成准确率提升或业务效果。
- 简历亮点可描述解析、分块、入库和评估流程，但将数据来源、分块策略、评估集与指标口径列为待确认。
- 面试风险应覆盖数据质量、切分策略、检索评估、答案评估、延迟/成本和失败回退。
- 高级版证据不足时明确降级，不堆叠“智能体、微调、大模型平台”等未出现的技术词。

## 失败信号

- 写成“主导 RAG 架构和模型选型”。
- 虚构准确率、召回率或用户满意度数字。
- 只润色措辞，不指出可被 AI 面试官追问击穿的证据缺口。
