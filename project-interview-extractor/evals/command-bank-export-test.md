# Eval：bank 全维度题库导出命令

## 输入

```text
$project-interview-extractor bank --role fullstack --level senior --coverage full --output docs/project-bank.md

这是一个 React + Node.js + PostgreSQL 的内部审批系统。我负责审批页面和部分 API，其他信息暂时不完整。
```

## 预期行为

- 在工作区创建 `docs/project-bank.md`，聊天中只返回摘要和文件链接。
- 文档覆盖矩阵包含全部 17 个维度，每个维度明确适用或 `N/A` 及理由。
- 所有适用维度至少包含 3 个主问题，高优先级维度至少包含 5 个。
- 每题包含意图、证据状态、回答要点、保守版回答、三层追问、踩坑和待确认项。
- 前端与后端维度都绑定审批系统事实，不虚构缓存、消息队列、生产流量或个人主导权。
- 文档包含术语地图、学习导读、跨维度场景题、学习路线和自测清单。
- 目标文件已存在且未传 `--overwrite` 时创建带时间后缀的新文件，不覆盖原文档。

## 失败信号

- 只在聊天中输出题库，没有写入文件。
- 用通用八股题凑满维度，没有关联审批流程。
- 缺少参考回答或把 `[待确认]` 写成既成事实。
- 宣称“所有问题”却没有覆盖矩阵和完成门槛。
