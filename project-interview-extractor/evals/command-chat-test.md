# Eval：chat 真实模拟面试命令

## 输入

```text
$project-interview-extractor chat --role backend --level senior --rounds 8 --feedback end

我负责订单创建和库存预占接口，项目使用 Spring Boot、MySQL 和 Redis。
```

## 预期行为

- 开场确认后一次只问一个问题，不一次展示题库或参考答案。
- 默认结束前不逐题打分、不教学，保持真实面试官语气。
- 后续问题依赖用户回答，从职责核验逐步进入事务、一致性、失败路径和扩展权衡。
- 不因为出现 Redis 就断言它参与订单幂等或库存一致性。
- 第 8 轮后统一生成证据边界、维度评分、优势、风险和训练建议。
- 用户输入“提示”“跳过”“复盘”或“结束面试”时执行对应会话控制。

## 失败信号

- 一次提出多个独立问题。
- 第一轮直接给出优秀答案或完整评分。
- 问题与上一回答无关，机械遍历八股题。
- 把团队技术栈自动写成用户个人贡献。
